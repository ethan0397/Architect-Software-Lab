Roadmap Học Software Architect

Đi từ nền tảng code-level design đến system design, tradeoff và architect mindset. Trọng tâm của lộ trình này là học theo vấn đề thực tế, không học rời rạc theo định nghĩa.

6 tháng 8-10 giờ / tuần Project-based learning

Mục lục

Toàn cảnh 5 nhóm năng lực Lộ trình 6 tháng Lịch học hằng tuần Portfolio projects Milestone tự đánh giá

Toàn cảnh

Software Architect không chỉ là người vẽ sơ đồ. Vai trò này cần hiểu business, biến nó thành hệ thống vận hành được, và giữ cho hệ thống vẫn thay đổi tốt sau nhiều vòng phát triển.

Giai đoạn Trọng tâm Kết quả mong đợi
Tháng 1-2 Nền tảng code và pattern Viết code rõ, dễ test, biết nhận ra vấn đề thiết kế
Tháng 3-4 Application architecture và distributed systems Thiết kế boundary tốt, hiểu consistency, retry, timeout
Tháng 5-6 Tradeoff, system design, architect mindset Chọn cấu trúc phù hợp và giải thích quyết định bằng ADR

Lộ trình 6 tháng

Tháng 1

Nền tảng kỹ thuật

OOP, SOLID, cohesion, coupling, clean code, refactoring, unit test và testability.

Tháng 2

Design Patterns cốt lõi

Strategy, Factory Method, Builder, Adapter, Facade, Observer, Command, Decorator, Repository, Dependency Injection.

Tháng 3

Application Architecture

Layered architecture, clean architecture, hexagonal architecture, modular monolith và CQRS basics.

Tháng 4

Data và Distributed Systems

Transaction, locking, cache aside, message queue, eventual consistency, idempotency, timeout, retry, circuit breaker.

Tháng 5

Architecture Styles và Tradeoff

Monolith, modular monolith, microservices, event-driven architecture, serverless, DDD strategic design.

Tháng 6

System Design và Architect Mindset

Non-functional requirements, capacity estimation, service boundaries, observability, security basics, architecture review.

Lịch học hằng tuần

Nhịp học gợi ý

  • 2 giờ lý thuyết
  • 3 giờ code ví dụ nhỏ
  • 2 giờ refactor code cũ
  • 1 giờ vẽ diagram
  • 1-2 giờ review và ghi note
Ghi nhớ Nếu chỉ đọc mà không vẽ hoặc không refactor, kiến thức sẽ khó bám. Công thức hiệu quả là: đọc -> code -> refactor -> vẽ -> review.

Milestone tự đánh giá

Sau 6 tháng bạn nên làm được

  • Phân biệt design pattern và architecture style
  • Giải thích tradeoff rõ ràng
  • Thiết kế boundary và dependency direction hợp lý
  • Nói được vì sao data consistency là vấn đề khó
  • Trình bày được một system design mạch lạc cho team
