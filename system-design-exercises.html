System Design Exercises

Mỗi bài tập giúp bạn luyện một vòng đầy đủ: làm rõ requirement, xác định NFR, ước lượng scale, vẽ data flow, chỉ ra bottleneck và nói rõ tradeoff.

Phương pháp luyện

BướcViệc cần làm
1Làm rõ requirement chức năng
2Làm rõ NFR
3Ước lượng scale
4Xác định component chính và data flow
5Chỉ ra bottleneck, failure mode, tradeoff
6Viết lại giải pháp như đang trình bày cho team
Mẹo luyện Đừng nhảy vào microservices quá sớm. Hãy hỏi dữ liệu đi đâu, ai sở hữu nó, và system lỗi ở đâu trước khi hỏi service nào tồn tại.

Bài tập

Case 1

URL Shortener

Học read/write pattern, key generation, caching, database indexing và analytics path.

Case 2

E-commerce Order Flow

Học service boundary, inventory consistency, payment integration và notification async.

Case 3

Booking System

Học concurrent booking, temporary reservation, expiration và availability view.

Case 4

Notification Platform

Học multi-channel delivery, provider failover, retry và template management.

Case 5

Audit Log System

Học immutable event, searchability, storage growth, security và compliance.

Checklist review

  • Boundary có rõ không?
  • Dependency có hợp lý không?
  • Data consistency được xử lý chưa?
  • Failure mode đã được nói tới chưa?
  • Monitoring/logging/metrics đã có chưa?
  • Có giải thích tradeoff không?

Tự chấm điểm

  • Rõ requirement: 1-5
  • Rõ NFR: 1-5
  • Boundary hợp lý: 1-5
  • Data flow rõ: 1-5
  • Nói được tradeoff: 1-5
  • Nói được failure mode: 1-5

Biến bài tập thành portfolio

Mỗi bài nên có

  • Problem statement
  • Assumptions
  • Context diagram
  • Container/component diagram
  • Sequence diagram
  • Database choice
  • Scaling strategy
  • ADR ngắn
Gợi ý Portfolio tốt không cần quá nhiều bài. Chỉ cần 2-3 bài nhưng phân tích sâu, có diagram sạch và tradeoff rõ là đủ mạnh.
