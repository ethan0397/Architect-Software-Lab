Each exercise helps you train a full loop: clarify requirements, identify NFRs, estimate scale, map the data flow, point out bottlenecks, and explain trade-offs.

Practice method

StepWhat to do
1Clarify functional requirements
2Clarify non-functional requirements
3Estimate scale
4Identify major components and data flow
5Call out bottlenecks, failure modes, and trade-offs
6Rewrite the solution as if you were presenting it to a team
Practice tip Do not jump into microservices too early. Ask where the data goes, who owns it, and where the system fails before asking which services exist.

Cases

Case 1

URL Shortener

Study read/write patterns, key generation, caching, database indexing, and analytics paths.

Case 2

E-commerce Order Flow

Study service boundaries, inventory consistency, payment integration, and asynchronous notifications.

Case 3

Booking System

Study concurrent booking, temporary reservation, expiration, and availability views.

Case 4

Notification Platform

Study multi-channel delivery, provider failover, retry strategies, and template management.

Case 5

Audit Log System

Study immutable events, searchability, storage growth, security, and compliance.

Review checklist

  • Are boundaries clear?
  • Are dependencies reasonable?
  • Is data consistency addressed?
  • Are failure modes discussed?
  • Are monitoring, logging, and metrics present?
  • Are trade-offs explained?

Score yourself

  • Requirement clarity: 1-5
  • NFR clarity: 1-5
  • Boundary quality: 1-5
  • Data flow clarity: 1-5
  • Trade-off quality: 1-5
  • Failure mode coverage: 1-5

Turning exercises into portfolio work

Each case should include

  • Problem statement
  • Assumptions
  • Context diagram
  • Container/component diagram
  • Sequence diagram
  • Database choice
  • Scaling strategy
  • A short ADR
Suggestion A strong portfolio does not need many cases. Two or three deep analyses with clean diagrams and explicit trade-offs are already powerful.
