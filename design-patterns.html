Creational Patterns

Factory Method

Gom logic tạo object

Dùng khi cần chọn implementation theo config hoặc môi trường.

Builder

Tạo object nhiều bước

Hợp với object có nhiều field optional hoặc quy trình dựng phức tạp.

Singleton

Một instance dùng chung

Hợp với config hoặc logger gốc, nhưng dễ dẫn tới global state nếu lạm dụng.

Structural Patterns

Adapter

Tích hợp vendor

Chuẩn hóa interface bên ngoài để không làm bẩn domain nội bộ.

Facade

Ẩn flow nhiều bước

Gom một use case lớn thành một điểm gọi rõ ràng.

Decorator

Thêm logging/cache/retry

Bọc object gốc mà không sửa class gốc.

Proxy

Kiểm soát truy cập

Lazy load, access control, remote proxy.

Repository

Bảo vệ domain khỏi ORM

Tạo boundary giữa business logic và persistence layer.

Behavioral Patterns

Strategy

Nhiều biến thể hành vi

Tách `if/else` dài thành từng chiến lược độc lập.

Observer

Một event, nhiều phản ứng

Publisher phát sự kiện, subscriber tự xử lý phần của mình.

Command

Đóng gói action

Rất hợp với queue, retry, undo/redo và deferred execution.

State

Hành vi theo trạng thái

Quản lý lifecycle như order, booking, payment.

Template Method

Khung thuật toán cố định

Hợp với import pipeline hoặc workflow có skeleton giống nhau.

Chain of Responsibility

Nhiều bước xử lý nối tiếp

Rất hợp với middleware, validation và approval flow.

Pattern theo loại hệ thống

Loại hệ thốngPattern hợpVì sao

Chọn pattern nhanh

Nếu bạn thấy...

  • Nhiều `if/else` theo biến thể hành vi -> Strategy
  • Logic `new` nằm rải khắp nơi -> Factory Method
  • Object có nhiều tham số -> Builder
  • API vendor làm bẩn domain -> Adapter
  • Use case gọi 5-7 service con -> Facade

Hoặc bạn thấy...

  • Muốn thêm cache/logging/retry -> Decorator
  • Object có nhiều trạng thái sống -> State
  • Request qua nhiều bước -> Chain of Responsibility
  • Action cần queue hoặc undo -> Command
  • Service dính ORM/SQL -> Repository
